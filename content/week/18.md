## The Are&period;na API

### Meet JSON

Before we get into our [Are.na](https://www.are.na) demo, let’s talk about [JSON](https://en.wikipedia.org/wiki/JSON) (*JavaScript Object Notation*)—the most common way an API responds with data:

- We’ll be using our [example channel](https://www.are.na/michael-fehrenbach/typography-and-interaction-too) we showed last week! Note the content as it is presented on Are&period;na.

- We can access the *data* from the channel by making a request to a special [Are&period;na API](https://www.are.na/developers/explore) URL, called [an “endpoint”](https://github.com/resources/articles/what-is-an-api#how-apis-work):

	[`https://api.are.na/v3/channels/typography-and-interaction-too`](https://api.are.na/v3/channels/typography-and-interaction-too)

- There the server responds with the channel content, now not as HTML, but as [*structured data*](https://en.wikipedia.org/wiki/Data_structure) in JSON—a hierarchical format/syntax that is based upon JavaScript, to represent different types of data and their relationships.

	In the context of a CMS, this structure is often reflects [the *schema*](https://en.wikipedia.org/wiki/Database_schema). <!-- .note -->

- Browsers don’t display JSON nicely by default, but there are extensions ([Chrome](https://json-formatter.js.org/), [Safari](https://apps.apple.com/us/app/json-peep-for-safari/id1458969831), [Firefox](https://addons.mozilla.org/en-US/firefox/addon/json_formatter/)) to make it easier to read.

- Changes made on Are&period;na will show up in the JSON the next time it is requested! In their schema, there is also a separate endpoint for the blocks themselves:

	[`https://api.are.na/v3/channels/typography-and-interaction-too/contents`](https://api.are.na/v3/channels/typography-and-interaction-too/contents)

- For the purposes of our project, we’ll only be *requesting* data (called a [`GET`&thinsp;](https://developer.mozilla.org/en-US/docs/Web/HTTP/Reference/Methods))—but know that APIs are also used to *send* (`POST`&hairsp;), *update* (`PUT`&hairsp;), and *delete* (…`DELETE`&hairsp;) as well.
